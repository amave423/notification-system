# Push Notification Service.

## Описание проекта

Push Notification Service — это микросервис, отвечающий за отправку уведомлений через push-канал в рамках распределенной системы уведомлений. Микросервис является независимым и масштабируемым компонентом, предназначенным для обработки push-уведомлений в асинхронном режиме.

## Архитектура

Микросервис построен по принципам чистой архитектуры с разделением на слои:
- **notification_sender** — бизнес-логика и контракты
- **models** — модели для внешнего взаимодействия
- **service** — веб-приложение с API и обработчиками сообщений
- **test** — модульные тесты

Для асинхронной обработки используется RabbitMQ: микросервис подписывается на очередь и обрабатывает сообщения из exchange `notifications-exchange` с routing key `push`.

## Технологии

- **Язык и фреймворк**: C# .NET 10
- **База данных**: PostgreSQL для хранения уведомлений и их статусов
- **Очереди сообщений**: RabbitMQ, для асинхронной обработки (aiopika)
- **Отправка push**: Клиенты, (пример: Pushbullet)
- **Тестирование**: pytest
- **Контейнеризация**: Docker Compose для локальной разработки (PostgreSQL, RabbitMQ, Elasticsearch)


## Запуск проекта

### Требования
- python 3.12.3
- Docker и Docker Compose

### Локальный запуск
1. Клонируйте репозиторий
2. Перейдите в директорию `src`
3. Запустите инфраструктуру: `docker-compose up -d`
4. Запустите приложение: `uvicorn system.service: app`

Приложение будет доступно на порту по умолчанию (8000).
